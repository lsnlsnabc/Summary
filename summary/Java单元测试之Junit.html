Java单元测试之Junit

Junit单元测试
基本单元测试步骤（基于Eclipse的Junit4）
  • 实现一个简单demo
public class Calculate {
     public int add(int a,int b){
           return a+b;
     }
     public int subtract(int a,int b){
           return a-b;
     }
     public int multiply(int a,int b){
           return a*b;
     }
     public int divide(int a,int b){
           return a/b;
     }
}
  • 导入Junit4的Jar包：BuildPath->Add Libraries->JUnit
  • 在包中新建CalculateTest new->JUnit Test Case
  • 在CalculateTest中编写测试代码快
public class CalculateTest {
     /*@Test
     public void test() {
           fail("Not yet implemented");
     }*/
     @Test //最简单的测试标注
     public void testAdd(){
           assertEquals(4, new Calculate().add(1,3));
     }
}
  • 选择Run As->Junit Test，运行测试代码
  • 简便操作：右键被测试程序，new->Junit Test Case，勾选需要的方法，生成文件

测试代码编写规则
  1. 测试方法上必须使用@Test进行修饰
  2. 测试方法必须使用public void 进行修饰，不能带任何参数
  3. 新建一个源代码目录来存放测试代码
  4. 测试类的包名应该和被测试类保持一致
  5. 测试单元中的每个方法必须可以独立测试，测试方法间不能有任何的依赖
  6. 测试类使用Test作为类名的后缀（非必须）
  7. 测试方法使用test作为方法名的前缀（非必须）

测试失败的两种情况
  • Failure一般由单元测试使用的断言方法判断失败所引起的，表示测试点发现了问题，程序输出结果和预期不一样
  • Error由代码异常引起，可以产生于测试代码本身错误，也可以是被测代码中的一个隐藏bug

Junit运行流程
  • @BeforeClass修饰的方法会在所有方法被调用前执行，该方法是静态的，当测试类加载后就会被运行，内存中它只会存有一份实例，比较适合加载配置文件
  • @AfterClass修饰的方法会在所有方法结束后执行，通常用来作资源清理，如关闭数据库连接
  • @Before和@After会在每个测试方法的前后各执行一次

Junit常用注解
  • @Test：将一个普通方法修饰成一个测试方法
  • @Test(expected=ArithmeticException.class):处理抛出的异常
  • @Test(timeout=毫秒):处理超时，可以作为性能测试
  • @Ignore:修饰方法被测试运行器忽略，可添加忽略的信息 Ignore("…")
  • @RunWith:更改测试运行器 org.junit.runner.Runner

Junit的各种断言
   断言是Junit提供的一些辅助函数，帮助我们确定被测方法是否按照预期的效果正常工作
   assertEquals([String message],expected,actual)
   assertTrue([String message],Boolean condition),验证给定的布尔型值是否为真
   assertNull([String message],Object object),验证给定对象是否为null
   assertSame([String message],expected,actual),验证expected参数和actual是否是同一个对象
   Fail([String message])

Junit测试套件的使用
  • 测试套件是组织测试类一起运行的测试类，写一个作为测试套件的入口类SuiteTest，这个类不包含其他的方法，更改测试运行器Suite.class，将要测试的类作为数组传入到Suite.SuiteClasses({})
package com.lsn.util;
import static org.junit.Assert.*;
import org.junit.Test;
import org.junit.runner.RunWith;
import org.junit.runners.Suite;
@RunWith(Suite.class)
@Suite.SuiteClasses({TaskTest1.class,TaskTest2.class})
public class SuiteTest {
     
}

Junit参数化设置
  • 更改默认的测试运行器为RunWith(Parameterized.class)
  • 声明变量来存放预期值和结果值
  • 声明一个返回值为Collection的公共静态方法，并使用@Parameters进行修饰
  • 为测试类声明一个带参数的公共构造函数，并在其中为之声明变量赋值
import static org.junit.Assert.*;
import java.lang.reflect.Parameter;
import java.util.Arrays;
import java.util.Collection;
import org.junit.Test;
import org.junit.runner.RunWith;
import org.junit.runners.Parameterized;
import org.junit.runners.Parameterized.Parameters;
@RunWith(Parameterized.class)
public class ParameterTest {
    int expected=0;
    int input1=0;
    int input2=0;
    @Parameters
    public static Collection<Object[]> t(){
        return Arrays.asList(new Object[][]{
            {3,1,2},
            {4,2,2}
        });
    }
    public ParameterTest(int expected,int input1,int input2){
        this.expected=expected;
        this.input1=input1;
        this.input2=input2;
    }
    @Test
    public void testAdd(){
        assertEquals(expected,new Calculate().add(input1,input2));
    }
}
